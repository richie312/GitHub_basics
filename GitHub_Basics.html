Basics of Git & Github

Linking Central Repository

  • To initialise local git repository on the hard drive: git init
  • Clear the gitbash screen clear
  • To link the local repository to the github origin git remote add origin "https://github.com/richie312/NLP.git"
  • To pull the files from the central repository into the local repository. git pull origin master

    • Git Commit

  • In order to know which files are already added and which are not? git status # "The file are which are in red indicates that are not yet added to the track of master origin."
  • To add the files on to the master repository, git commit -a -m "Files added by Richie @ 4th Apr"
  • To get the history of commit git log
    • git add filename
  • Pushing the files from the branches to Central Repository (Github Repository). git push origin
    • Revert to older files just go to git log and paste the first 8 hexadecimal digits along with the files in order to revert in the following manner,
    git check out
    Check inside of the file with the following command,
    cat recert.R

    Git Reset

    if things are messed up and you want to reset to the older version of the repository then run the following instructions, Note: commit is the alpha numeric string which you will get from git log

    • git reset --hard 1c4e3847a13a9420315a924968cf564127089cdc
    • git reset --hard origin/master

    Merging

  • in order to look for the files in different branches, ls
  • In order to create a branch git branch firstbranch
  • To switch to the other branch, git checkout firstbranch
  • 12. Merge the branches git merge ## All the newly created files and folder in that branch will now be updated on to the other branch as well.
  • Rebase the branches Created two more files in first branch and then went to master branch in order to rebase the firstbranch git rebase firstbranch
